# 3. 아키텍처 기준

> 상세: TDD §3, §6 / CLAUDE.md §1~§2

**4계층 DAG** (확정, ESLint 적용 완료)

```
app/ (Composition Root)
  → server/ + client/ (비즈니스)
    → shared/ (순수 타입/상수/유틸)
```

**3앱 통합 구조** (TDD §6 기반 확장)

```
app/[locale]/*   → 사용자 앱
app/admin/*      → 관리자 앱
app/partner/*    → 제휴업체 앱 (v0.2)
app/api/*        → 공통 API (사용자/관리자/제휴업체 하위 분리)

server/core/     → 3앱 공유 시스템 인프라 (AI, DB, config)
server/features/ → 비즈니스 코드 (chat, beauty, admin, partner 등)
client/core/     → 공유 UI 인프라
client/features/ → 비즈니스 UI (chat, cards, admin 등)
shared/          → 순수 타입, 상수, 유틸
```

**핵심 원칙**: P-1(DAG), P-2(Core 불변), P-3(Last Leaf), P-4(Composition Root), P-5(콜 스택 ≤4)
