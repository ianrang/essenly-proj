# 1. 제품 개요

## 1.1 사업 정체성

에센리 K-뷰티 AI 에이전트 — K-뷰티 의사결정 보조 웹 앱.

에센리(Essenly)의 미션은 한국을 방문하는 외국인 여성 여행객이 K-뷰티 관련 선택(제품, 시술, 살롱, 맛집, 문화 체험)을 쉽고 정확하게 내릴 수 있도록 AI 기반 가이드를 제공하는 것이다.

## 1.2 핵심 문제

한국 방문 외국인 여행객이 겪는 K-뷰티 정보 비대칭:
- 언어 장벽: 제품·시술 정보가 주로 한국어. 영문 리뷰/가이드 부족
- 선택 과부하: 수천 개 제품, 수백 개 클리닉 — 본인에게 맞는 것을 판별하기 어려움
- 맥락 부재: 피부 타입, 여행 일정, 예산에 맞는 통합 정보 없음
- 신뢰 문제: 광고성 추천과 진정한 추천 구분 어려움

## 1.3 솔루션

AI 기반 대화형 에이전트:
- 다국어 대화: 6개 언어로 자연스러운 대화
- 개인화 추천: 피부 타입, 고민, 일정, 예산 기반
- 카드 UI: 대화 안에 구조화된 추천 카드 (하이브리드 인터페이스)
- 비개입적 판단: 광고비가 추천 순위/선정에 영향을 주지 않음

## 1.4 대상 사용자

| 속성 | 정의 |
|---|---|
| 인구통계 | 20~40대 여성 |
| 국적 | 한국 외 전 세계 |
| 상황 | 한국 방문 예정 또는 방문 중 여행객 |
| 관심사 | K-뷰티 제품 구매, 시술, 살롱, 뷰티 관련 문화 체험 |

## 1.5 핵심 가치 원칙

| ID | 원칙 | 설명 |
|---|---|---|
| VP-1 | 비개입적 판단 | 광고비·제휴 수수료가 추천 순위·선정에 개입 불가. 하이라이트(시각적 강조)만 허용. 순위 변경·우선 노출·필터 상위 배치 금지. |
| VP-2 | 국가 독립적 설계 | 데이터 모델은 한국 한정이 아닌 글로벌 확장 가능 구조. ISO 표준 사용 (국가, 통화, 언어). |
| VP-3 | 점진적 개인화 | 정보가 없어도 서비스 가능. 정보가 쌓일수록 정밀해지는 구조. |
| VP-4 | 대화 + 카드 하이브리드 | 자연어 대화 안에 구조화된 추천 카드를 삽입하는 인터페이스. |

# 2. 서비스 범위

## 2.1 5영역 도메인

| ID | 영역 | 설명 |
|---|---|---|
| DOM-1 | 쇼핑 (Shopping) | K-뷰티 제품 추천 + 구매 가능 매장 안내 |
| DOM-2 | 시술 (Treatment/Clinic) | 피부과/미용 시술 추천 + 클리닉 안내 |
| DOM-3 | 살롱 (Salon) | 헤어살롱, 헤드스파, 네일/속눈썹 등 |
| DOM-4 | 로컬 맛집 (Dining) | 뷰티 여정 중 식사/카페 — 여행 연계 |
| DOM-5 | 문화 체험 (Experience) | K-뷰티 관련 문화 활동 — 여행 연계 |

## 2.2 Action Layer 3단계

| Stage | 기능 | 설명 |
|---|---|---|
| Stage 1 (기반) | F1: 정보 제공, F2: 외부 링크 연결 | 제품/장소 정보 + 외부 지도·구매·예약 링크 |
| Stage 2 (차별화) | F3: 뷰티 여정 코스, F7: 자동 팔로업 | 다일 코스 생성 + 행동 기반 후속 추천 |
| Stage 3 (외부 연동) | F4: 실시간 예약, F5: 알림, F6: 리뷰 수집 | 파트너십 기반 예약·알림·리뷰 시스템 |

> 기능 번호는 정의 순서이며, Stage 순서와 무관하다.

## 2.3 수익 구조

| 수익원 | 설명 | 비개입적 원칙 준수 |
|---|---|---|
| 에센리 키트 판매 | AI 프로필 기반 맞춤 K-뷰티 키트 | 에센리 제품은 하이라이트 카드로만 시각적 구분. 추천 순위 불변. |
| 제휴 수수료 | 외부 링크 클릭 → 제휴 수익 (Stage 1) | 링크 존재 여부가 추천 선정에 영향 없음 |
| 프리미엄 광고 | 하이라이트 배지 판매 (후기) | 시각적 강조만. 정렬 미영향. |
| B2B 리포트 | 외국인 뷰티 소비 데이터 분석 (후기) | 비식별 집계 데이터 |

## 3.1 전체 흐름도

```
┌──────────────────────────────────────────────────────────┐
│                     CORE FLOW (MVP)                       │
│                                                           │
│  [Entry] ──> [Landing] ──> [동의] ──> [Chat]             │
│                                        ├─ 채팅 내 온보딩  │
│                                        └─ 카드 추천       │
├──────────────────────────────────────────────────────────┤
│                    CONVERSION LAYER                       │
│                                                           │

```

> **MVP Chat-First 단일 경로**: Landing → Chat → 동의 → 채팅 시작. 온보딩은 채팅 내 AI 대화로 수행 (VP-3). Onboarding 폼/Profile 페이지는 v0.2(이메일 로그인) 범위. 상세: `mvp-flow-redesign.md`

## 3.2 Landing 화면

### 화면 구성

Landing은 **서비스 소개 마케팅 페이지 + 앱 진입점**을 겸한다. 섹션 스크롤 방식으로 서비스 가치를 전달한 후 행동을 유도한다.

**레이아웃**: 풀 너비 반응형. 앱 페이지(640px 중앙)와 **별도 레이아웃**을 사용한다.
- 모바일(기본): 단일 컬럼, 풀 너비
- 데스크톱(lg:): 풀 너비, 섹션별 max-width 제한 + 중앙 정렬

```
┌────────────────────────────────────┐
│  [Essenly 로고]        [EN ▾]      │  ← Header
├────────────────────────────────────┤
│  Section 1: Hero + CTA             │
│  "Discover K-Beauty, Tailored..."  │
│  [Start chatting]                  │  ← 단일 CTA (Chat 직행)
│  "Chat with our AI guide —         │
│   no signup needed"                │
├────────────────────────────────────┤
│  Section 2: How it works           │
│  ① Chat with our AI guide          │
│  ② Get AI-matched picks            │
│  ③ Shop & book instantly           │
├────────────────────────────────────┤
│  Section 3: What you get           │
│  맞춤 상품 / 검증 클리닉 /          │
│  지도+예약 / 무료 키트 / 익명 무료   │
├────────────────────────────────────┤
│  Section 4: Trust                  │
│  개인정보 보호 3항목 (CTA 없음)     │
└────────────────────────────────────┘
```

> **CTA 중복 없음**: CTA는 Section 1(Hero)에만 1회 배치. Section 4는 신뢰 항목만 표시.

> **Hero 배경**: CSS 그래디언트 애니메이션. `prefers-reduced-motion` 시 정적 배경으로 전환.

> **언어 선택**: 언어 코드+원어명 병기 (`EN English` 등). 국기 아이콘 미사용 (언어≠국가, 정치적 리스크).

> **데스크톱 앱 목업**: MVP에서는 미리보기 이미지를 포함하지 않는다 (채팅 UI 미구현 상태에서 가짜 스크린샷은 기대값 불일치). v0.2에서 실제 앱 스크린샷 추가 예정.

### 분기 로직

```
Landing 진입
  │
  ├─ 세션 존재? ─── YES ──> [재방문 흐름]
```

> **v0.2 범위**: "Set up my profile" 보조 CTA + Onboarding 폼 + Profile 페이지는 이메일 로그인 도입 시 활성화. `mvp-flow-redesign.md` §3 참조.

### 데이터 수집
- RT-2 (현재 시간): 자동
- UP-3 (언어): 자동 감지 → 대화 언어 선택 UI (MVP: UI는 영어 고정, 대화 언어만 선택)
- 프로필 존재 여부: 저장된 프로필 존재 여부 확인

### 동의 수집
- Chat 진입 시 인라인 동의 확인: 데이터 보관 동의를 Chat 화면에서 수집 (CTA → /chat → ConsentOverlay → 동의 확인 → 세션 생성)
- 마케팅 동의는 Kit CTA 이메일 제출 시점에 별도 체크박스로 수집
- MVP 필요 동의: 장기 보관(필수) + 마케팅(Kit CTA 시점). 위치/행동로깅은 MVP 제외

### 재방문 판별 실패
- 식별 정보(쿠키/anonymous UUID) 소실 시 신규 사용자로 처리
- 이전 데이터는 서버에 잔존하나 연결 불가. 복구는 v0.2(계정)에서 제공

## 3.5 카드 UI 요구사항

### 카드 공통 구조

모든 추천 카드의 공통 요소:
- 헤더: 이름 + 가격/가격대
- 본문: 추천 이유 (AI 생성 — "왜 이것을 추천하는지" 개인화된 설명)
- 하단: 언어 지원 표시 · 위치/지도 링크 · 액션 버튼
- 하이라이트: 선택적 배지 표시 (VP-1: 시각적 강조만, 순위 미영향)

### 도메인별 카드

| 카드 타입 | 도메인 | 핵심 표시 정보 |
|---|---|---|
| ProductCard | DOM-1 쇼핑 | 제품명, 브랜드, 가격, 추천 이유, 영문 라벨 유무, 구매 위치 |
| TreatmentCard | DOM-2 시술 | 시술명, 클리닉, 가격대, 소요 시간, 회복 기간, 적합성 |
| SalonCard | DOM-3 살롱 | 살롱명, 서비스 목록, 가격대, 분위기, 언어 지원 |
| DiningCard | DOM-4 맛집 | 맛집명, 요리 유형, 가격대, 분위기, 영문 메뉴 유무, 뷰티 맥락 |
| ExperienceCard | DOM-5 체험 | 체험명, 유형, 가격, 소요 시간, 가능 언어, 뷰티 연결점 |

> 하이라이트 카드: 별도 카드 타입이 아닌, 모든 카드에 적용 가능한 시각적 강조 래퍼. 도메인 엔티티의 하이라이트 속성에 의해 결정 (§4-B 공통 요구사항 참조).

### 카드 내 매장/클리닉 표시 규칙

제품↔매장, 시술↔클리닉은 다대다 관계. 카드에는 AI가 대화 맥락(사용자 언급 지역, 위치, 선호 조건)에 따라 가장 적합한 1개를 선택하여 표시. 맥락 없을 시 접근성/인기순 폴백. ProductCard와 TreatmentCard에 동일 규칙 적용.

### 가격 표시 통화

- MVP: KRW(₩) 단일 표시. 한국 현지 가격 기준.
- 환율 변환(USD 병기 등)은 v0.2 이후 검토.

# 4. 데이터 요구사항

## 4-A. 개인화 체계

4계층 15개 변수를 수집·관리하여 개인화된 추천을 제공한다.

```
┌─────────────────────────────────────────────────┐
│              사용자 데이터 모델                    │
│                                                  │
│  ① 사용자 프로필 (4개)  ─ 반영구적               │
│  ② 여정 컨텍스트 (5개)  ─ 방문마다 변경           │
│  ③ 뷰티 히스토리 (4개)  ─ 시간 누적               │
│  ④ 실시간 컨텍스트 (2개) ─ 자동 수집              │
│                                                  │
│         ↓ AI 추론 ↓                               │
│                                                  │
│  [도출 변수 4개] 선호/기피 성분, 세그먼트, 프로필   │
└─────────────────────────────────────────────────┘
```

### ① 사용자 프로필 (User Profile) — 반영구적

| ID | 변수 | 분류 / 허용 값 | 필수 | 수집 | 사용 목적 |
|---|---|---|---|---|---|
| UP-1 | 피부 타입 (`skin_types`) | 건성 · 지성 · 복합성 · 민감성 · 보통 (5개, **배열 max 3**). 복합 피부(건성+민감 등) 현실 반영을 위해 다중값 허용 (NEW-17, 2026-04-15). 온보딩 UI에서 최대 3개 선택, 대화로 AI 추출 시 기존값과 합집합 병합. 허용값: dry/oily/combination/sensitive/normal | 권장 | 온보딩 / 대화 | 제품 추천 필터, 시술 적합성, 성분 판단 |
| UP-2 | 헤어 상태 | 타입(직모 · 웨이브 · 곱슬 · 코일리) + 고민(손상 · 탈모 · 지성두피 · 건조끝 · 비듬 · 염색, 6개, 복수) | 선택 | 온보딩 / 대화 | 헤어 제품 · 살롱 추천 |
| UP-3 | 국가 / 언어 | 국가: ISO 3166-1 alpha-2. 언어: 6개(en · ja · zh · es · fr · ko) | 필수 | 자동 감지 → 온보딩 확인 | UI 언어, 대화 언어, 외국인 매칭 |
| UP-4 | 연령대 | 18-24 · 25-29 · 30-34 · 35-39 · 40-49 · 50+ (단일) | 선택 | 온보딩 / 대화 | 세그먼트 도출, 적합성 판단 |

> 피부 타입에 '보통(normal)' 포함 — 5개 허용값, 최대 3개 선택 (배열). UI 라벨 "Normal / Not sure"로 미확정 사용자도 수용.

### ② 여정 컨텍스트 (Journey Context) — 방문마다 변경

| ID | 변수 | 분류 / 허용 값 | 필수 | 수집 | 사용 목적 |
|---|---|---|---|---|---|
| JC-1 | 피부 고민 | 11개 풀: 여드름 · 주름 · 잡티 · 홍조 · 건조 · 모공 · 칙칙함 · 다크서클 · 톤불균형 · 자외선손상 · 습진 (복수) | 권장 | 온보딩(최대 3개) / 대화(최대 5개 보관) | 제품 · 시술 매칭, 성분 판단 |
| JC-2 | 관심 활동 | shopping · clinic · salon · dining · cultural (5개 = 도메인 1:1 매핑, 복수) | 권장 | 온보딩 / 대화 | 관심 도메인 필터, 코스 구성 |
| JC-3 | 체류 기간 | 일수(1~30) + 시작일/종료일(선택) | 권장 | 온보딩 / 대화 | 시술 회복기간 적합성(핵심), 코스 일수 |
| JC-4 | 예산 | 저가 · 중가 · 프리미엄 · 럭셔리 (4단계, 단일) | 선택 | 온보딩 / 대화 | 가격대 필터 |
| JC-5 | 여행 스타일 | 효율적 · 여유 · 모험 · 인스타 · 로컬경험 · 럭셔리 · 알뜰 (7개, 복수) | 선택 | 온보딩 / 대화 | 코스 밀도, 장소 분위기 매칭 |

> 피부 고민 — 전체 풀 11개. 온보딩 UI에 7개 표시(여드름·주름·잡티·홍조·건조·모공·칙칙함), 최대 3개 선택. 나머지 4개(다크서클·톤불균형·자외선손상·습진)는 대화 추출. 최대 5개 보관. 설정 기반 관리. 점진적 개인화(VP-3).
> 여행 스타일 — 전체 풀 7개. 온보딩 UI에 5개 표시(효율적·여유·모험·인스타·로컬경험). 나머지 2개(럭셔리·알뜰)는 대화 추출, JC-4(예산)과 중첩되므로 온보딩 제외. 설정 기반 관리.
> 예산 — 단계(level)만. 금액/통화는 v0.2 필요 시 추가.
> 관심 활동 — 5개, 도메인 1:1 매핑. 설정 기반 관리.

### ③ 뷰티 히스토리 (Beauty History) — 시간 누적

| ID | 변수 | 기록 내용 | 수집 | 사용 목적 |
|---|---|---|---|---|
| BH-1 | 시술 이력 | 시술명, 클리닉, 일자, 만족도(1-5), 메모 | 대화 / 예약 후 자동(Stage 3) / 수동 | 중복 시술 방지, 관련 추천, 사후 관리 |
| BH-2 | 구매 이력 | 제품명, 매장, 일자, 만족도(1-5), 메모 | 대화 / Kit CTA 전환 시 / 수동 | 유사 제품 추천, 만족 제품 재추천 |
| BH-3 | 방문 이력 | 장소명, 장소 유형(매장/클리닉/살롱/맛집/체험), 일자, 만족도(1-5) | 위치 기반(동의 시) / 대화 / 수동 | 재방문 · 유사 장소 추천, 코스에서 방문 완료 제외 |
| BH-4 | 학습된 선호도 | 카테고리, 선호 항목, 방향(선호/비선호), 확신도, 출처 | 대화 분석 / 행동 패턴 / 만족도 | 도출 변수(DV-1, DV-2) 계산 입력 |

> BH-4 갱신 규칙: 새 데이터가 기존과 충돌 시 최신 데이터 우선 + 확신도 조정.

### ④ 실시간 컨텍스트 (Real-time Context) — 자동 수집

| RT-1 | 현재 위치 | 위치 정보 수집 (사용자 동의 필수). 미동의 시 지역 직접 입력 유도 | 근처 장소 추천, 코스 최적화, 이동 시간 |
| RT-2 | 현재 시간 | 자동 수집 (타임존 포함) | 영업 중 장소 필터, 시간대별 추천 |

### 도출 변수 (Derived Variables)

수집 변수를 기반으로 실시간 계산.

| ID | 명칭 | 입력 | 산출 로직 | 용도 |
|---|---|---|---|---|
| DV-1 | 선호 성분 | UP-1 + JC-1 + BH-4(선호) | 피부타입 + 고민 → 성분 매핑 + 학습 가중치 | 제품 추천 우선 필터 |
| DV-2 | 기피 성분 | UP-1 + BH-4(비선호) | 피부타입 → 주의 성분 + 명시적 비선호 | 제품 제외 필터 |
| DV-3 | 사용자 세그먼트 | UP-4 + JC-2 + JC-4 + JC-5 | 규칙 기반 분류 | 마케팅 · 분석용 (추천에 직접 미사용) |
| DV-4 | AI 뷰티 프로필 | 모든 변수 + DV-1~3 | AI 종합 프로필 생성 (자연어 요약 + 구조화) | 프로필 화면 표시, AI 추천 컨텍스트 |

### 시술 추천 규칙 (downtime 적합성)

- 시술 회복기간(downtime)이 잔여 체류일보다 길면 해당 시술을 추천에서 제외
- 회복기간이 잔여 체류일의 50% 이상이면 경고 표시
- 잔여 체류일 계산: 종료일이 있으면 종료일 - 오늘, 없으면 체류 일수를 잔여 일수의 폴백으로 사용 (과대평가 가능성 있음 — 대화에서 정확한 일정 추가 확인 권장)

## 4-B. 도메인 데이터 요구사항

### 공통 요구사항

모든 엔티티 공통:
- 다국어 텍스트 지원 (6개 언어: 한국어 원본 + 영어 · 일본어 · 중국어 · 스페인어 · 프랑스어). 언어 추가 시 엔티티 구조 변경 없어야 한다
- 하이라이트(시각적 강조) 적용 가능 — 배지 텍스트 표시. VP-1 비개입적 판단 원칙: 추천 순위·선정에는 미영향
- 평점(0.0–5.0), 리뷰 수, 이미지, 태그, 상태 관리(활성 / 비활성 / 임시 휴업)

장소형 엔티티 추가:
- 위치 정보: 좌표, 주소(다국어), 국가(ISO 3166-1), 도시, 행정구역
- 영업시간 (요일별 + 휴무일 + 특이사항)
- 외국인 지원 수준: 없음 / 기본 / 유창
- VP-2 국가 독립적 설계: 한국 한정이 아닌 글로벌 확장 가능 구조

> 비장소형 엔티티(제품, 시술)에는 위치 정보가 불필요하므로, 공통 속성과 장소 속성을 분리한다.
> 리뷰 요약(AI 생성, 다국어): MVP에서는 Products에만 적용. Store/Clinic/Treatment 등 전체 확장은 v0.2에서 검토.

### DOM-1 쇼핑 (Shopping)

제품 (Product) — 비장소형
- 기본: 제품명(다국어), 브랜드(참조), 이미지
- 분류: 카테고리(스킨케어 · 메이크업 · 헤어케어 · 바디케어 · 도구), 세부 카테고리
- 개인화 매칭: 적합 피부타입, 적합 헤어타입, 대응 고민, 핵심 성분(참조), 주의 성분(참조)
- 구매: 가격(KRW), 용량, 구매 가능 매장(참조), 온라인 구매 링크
- 관광객: 영문 라벨 유무, 관광객 인기 여부
- AI 생성: 리뷰 요약(다국어)

매장 (Store) — 장소형
- 유형: 올리브영 · 시코르 · 백화점 · 브랜드 직영 · 약국 · 기타
- 취급 브랜드, 관광객 서비스(면세 · 할인 · 샘플바 등), 결제 수단, 인근 랜드마크

브랜드 (Brand)
- 원산지, 가격 포지셔닝(럭셔리 · 중가 · 저가 · 인디), 에센리 자사 여부, 전문 분야

성분 (Ingredient)
- INCI 표준명(다국어), 기능, 주의 필요 피부타입, 주로 포함되는 카테고리

### DOM-2 시술 (Treatment/Clinic)

클리닉 (Clinic) — 장소형
- 유형: 피부과 · 성형외과 · 에스테틱 · 메드스파
- 면허 확인 여부, 의사 정보(전문 분야 · 가능 언어 · 자격), 상담 유형(대면 · 온라인 · 전화)
- 외국인 지원 상세: 상담 가능 언어, 통역 서비스, 영문 동의서, 해외 카드 결제, 픽업 서비스
- 제공 시술 목록(참조), 예약 링크, 외부 링크

시술 (Treatment) — 비장소형
- 분류: 카테고리(피부 · 레이저 · 주사 · 페이셜 · 바디 · 헤어), 세부 카테고리
- 개인화 매칭: 대응 고민, 적합 피부타입
- 일정 연계: 가격대, 소요 시간(분), 회복 기간(일) — §4-A 시술 추천 규칙과 직결, 권장 횟수
- 안전: 주의사항(다국어), 사후 관리 안내(다국어)
- 제공 클리닉(참조)

### DOM-3 살롱 (Salon)

살롱 (Salon) — 장소형
- 유형: 헤어 · 헤드스파 · 네일 · 속눈썹 · 복합
- 서비스 목록, 스타일리스트 정보(전문 분야 · 가능 언어 · 포트폴리오)
- 외국인 지원 상세, 분위기 태그, 예약 링크, 외부 링크

살롱 서비스 (SalonService)
- 분류: 커트 · 컬러 · 펌 · 트리트먼트 · 헤드스파 · 네일 · 속눈썹 · 두피케어
- 적합 헤어타입, 가격대, 소요 시간

### DOM-4 로컬 맛집 (Dining)

맛집 (Restaurant) — 장소형
- 유형: 한식 · 카페 · 디저트 · 퓨전 · 길거리음식 · 파인다이닝
- 분위기 태그(인스타 · 숨은맛집 · 현지인기 · 관광객친화)
- 1인 가격대, 추천 메뉴(메뉴명 · 가격 · 설명 · 이미지), 식이 옵션(채식 · 할랄 · 글루텐프리 등)
- 영문 메뉴 유무, 예상 대기시간, 예약 필요 여부
- 뷰티 여정 연계: 인근 뷰티 장소(참조), 맥락 태그(시술 후 친화 · 피부 음식 · 인스타 카페 등)

### DOM-5 문화 체험 (Experience)

체험 (Experience) — 장소형
- 유형: 클래스 · 투어 · 워크숍 · 팝업 · 전시 · 마켓
- 뷰티 관련성: 직접 · 관련 · 라이프스타일
- 카테고리(스킨케어 클래스 · K-뷰티 투어 · 템플 스파 · 한복 · K-pop 등)
- 가격대, 소요 시간, 인원, 예약 필수 여부, 체험 가능 언어
- 일정(유형: 매일 · 주간 · 특정일 · 주문형 + 시간대 + 불가일)
- 포함 사항, 뷰티 여정과의 연결점 설명(다국어), 외부 링크

### 교차 도메인 관계

- 제품 ↔ 매장: 구매 가능 매장 연결
- 제품 ↔ 브랜드: 브랜드 정보 연결
- 제품 ↔ 성분: 핵심/주의 성분 연결
- 클리닉 ↔ 시술: 제공 시술 목록 연결
- 클리닉 ↔ 의사: 소속 의사 연결
- 살롱 ↔ 서비스: 제공 서비스 연결
- 살롱 ↔ 스타일리스트: 소속 스타일리스트 연결
- 맛집 ↔ 뷰티 장소: 인근 매장/클리닉/살롱 연결 (뷰티 여정 맥락)
- 지역 기반 교차 검색: 모든 장소형 엔티티의 위치 · 행정구역 활용

> 모든 장소형 엔티티에 외부 링크(지도 · 웹사이트 · 인스타그램 · 예약 등) 적용. 제품은 온라인 구매 링크 별도 관리.
> 클리닉 · 살롱은 외국인 지원 상세 정보(언어 · 통역 · 동의서 · 결제 · 픽업) 관리. 맛집은 간소화(영문 메뉴 유무 수준). 체험은 체험 가능 언어로 대응.
