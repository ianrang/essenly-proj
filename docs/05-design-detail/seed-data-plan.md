# §1 목적 + 범위

## 1.1 목적

서비스 가동에 필요한 두 종류의 데이터 준비 계획을 정의한다:

## 2.2 수집 방법별 역할

data-strategy.md U-6, U-8 결정 기반.

### 2.2.1 API 자동 수집 (stores, clinics)

| 소스 | 대상 | 수집 필드 | 누락 → 수동 보완 |
|------|------|----------|----------------|
| 카카오 로컬 API | stores | name.ko, address.ko, location, phone, store_type | 영업시간, english_support, tourist_services, description |
| 카카오 로컬 API | clinics | name.ko, address.ko, location, phone, clinic_type | english_support, foreigner_friendly, consultation_type, description |

수집 쿼리 예시:
- "강남 올리브영", "명동 화장품", "홍대 뷰티" (stores)
- "강남 피부과", "명동 클리닉", "압구정 성형외과" (clinics)

### 2.2.2 수동 입력 (products, treatments, brands, ingredients)

제품 데이터는 A-3 시드 크롤링(브랜드 사이트 1순위 + 올리브영 글로벌 2순위) + CSV 수동 + AI 보강. 시술은 수동 입력이 주 방법.

| 입력 방식 | 적용 대상 | 설명 |
|----------|----------|------|
| A-3 시드 크롤링 (Playwright) | products | 브랜드 공식 사이트(1순위) + 올리브영 글로벌(2순위 보조). name_en, brand, price, category, image 자동 수집 |
| 관리자 앱 CRUD | 모든 엔티티 | 7.2-ADMIN-REQUIREMENTS.md 기반 |
| CSV 일괄 업로드 | products, ingredients, treatments | 초기 대량 입력 + 크롤링 미대상 제품 보완 |
| ~~쿠팡 파트너스 API (S7)~~ | ~~products~~ | **MVP 보류** (U-12, 판매 실적 미달). 수익 발생 후 재검토 |
| AI 보강 | description, review_summary, 번역, 분류 | LLM 처리 후 전수 검수 (D-7) |

### 2.2.3 AI 보강 단계

| 보강 대상 | AI 역할 | 검수 필요 |
|----------|---------|----------|
| name.en (+ ja, zh, es, fr) | LLM 번역 (P0-14 검증) | 샘플 10% 수동 검수 |
| description.ko | 제품/시술 특성 기반 설명 생성 | 전수 검수 |
| review_summary | AI 생성 요약 (U-14 결정) | "AI 기반" 면책 확인 |
| ingredients (1순위 소스) | 식약처 원료성분(S3) + CosIng(S6) 기반 수집 | 전문가 검수 |
| ingredients.function | 성분 기능 설명 | 전문가 검수 |
| ingredients.caution_skin_types | 피부타입별 주의 성분 | 전문가 검수 |

# §3 뷰티 지식 KB (P1-62)

## 3.1 KB 범위

PRD §4-B 정의 기반. LLM이 대화 중 참조하는 도메인 전문 지식.

| 지식 유형 | 용도 | 예시 내용 | 우선순위 |
|----------|------|----------|---------|
| 성분 가이드 | 피부타입별 추천/주의 성분 판단 | "나이아신아마이드: 미백+보습. 민감성 피부 저농도 권장" | **1순위** (DV-1, DV-2 판단 근거) |
| 시술 가이드 | 적합 피부타입, 회복기간 기반 추천 | "레이저 토닝: 색소 치료. 다운타임 3~5일. 건성 피부 주의" | **1순위** (검색·판단에 직접 사용) |
| 지역 가이드 | 서울 주요 뷰티 지역 특성 안내 | "강남: 프리미엄 클리닉 밀집. 압구정: 성형·피부과 특화" | 2순위 |
| K-뷰티 상식 | 문화, 쇼핑 팁, 에티켓 | "한국 화장품 매장에서 테스터 사용 에티켓" | 2순위 |

### 1순위 상세 범위

**성분 가이드** (ingredients KB):
- 주요 활성 성분 30~50종
- 각 성분: 기능, 효과, 피부타입별 적합도, 주의사항, 시너지 성분
- DB ingredients 테이블과 연계 (inci_name으로 매칭)

**시술 가이드** (treatments KB):
- 주요 시술 20~30종 (DB treatments 엔티티와 1:1 또는 1:N 대응)
- 각 시술: 원리, 적합 피부타입, 다운타임, 시술 횟수, 비용대, 주의사항
- DB treatments 테이블의 precautions, aftercare와 상호 보완 (중복 아닌 심화)

## 3.2 작성 주체 결정

| 방식 | 장점 | 단점 | 비용 | 속도 |
|------|------|------|------|------|
| 전문가 직접 작성 | 최고 정확도 | 비용 높음, 속도 느림 | $$$ | 느림 |
| **AI 초안 + 전문가 검수** | 비용·속도 최적 | 검수 필요 | $ | 빠름 |
| AI 단독 | 최저 비용 | 의학적 정확성 우려 | $ | 최빠름 |

**MVP 결정: AI 초안 + 전문가 검수**

- AI (Claude/Gemini)가 초안 작성
- K-뷰티 도메인 전문가 1인이 검수
- 성분 가이드: 의학적 정확성 필수 → 검수 중점
- 시술 가이드: 의학적 정확성 필수 → 검수 중점
- 지역/상식 가이드: AI 품질로 충분 → 가벼운 검수

### 3.3.2 문서 구조 (성분 예시)

```markdown
# 나이아신아마이드 (Niacinamide)

## 기본 정보
- INCI: Niacinamide
- 별명: 비타민 B3
- 기능: 미백, 보습, 피지 조절

## 피부타입별 적합도
- 지성: 매우 적합 (피지 조절)
- 건성: 적합 (보습 효과)
- 민감성: 저농도(5% 이하) 권장
- 복합성: 매우 적합

## 시너지 성분
- 히알루론산: 보습 시너지
- 레티놀: 교차 사용 권장 (동시 사용 시 자극 가능)

## 주의사항
- 비타민 C와 동시 사용 시 효과 상충 (교차 사용 권장)
- 고농도(10%+) 사용 시 홍조 가능
```

### 3.3.3 임베딩 연계 (P1-38)

- 각 Markdown 파일 = 1 임베딩 청크 후보
- 긴 문서는 섹션 단위로 분할 (P1-38에서 청크 전략 확정)
- 파일명 = 검색 시 참조 ID

## 3.4 품질 기준

| 기준 | 성분/시술 (1순위) | 지역/상식 (2순위) |
|------|-----------------|-----------------|
| 의학적 정확성 | 전문가 검수 필수 | 해당 없음 |
| 문화적 적절성 (외국인 관점) | 외국인 UX 검수 | 외국인 UX 검수 |
| 다국어 | ko 원본 → en 번역 필수 | ko 원본 (en 번역 선택) |
| 완성도 | 모든 섹션 작성 | 핵심 정보만 |

### 다국어 KB 전략

- **MVP**: ko 원본으로 작성 → LLM이 사용자 언어로 실시간 응답 생성
- KB 자체를 6개 언어로 번역하지 않음 (LLM 컨텍스트이므로 ko면 충분)
- LLM이 ko KB를 읽고 en/ja/zh/es/fr로 응답하는 방식 (P0-14에서 검증)

## 3.5 타임라인

AI 통합 테스트 전 1순위 KB 완료 필수.

| 마일스톤 | 목표 | 범위 | 수량 |

| 검수 대상 | 검수자 | 검수 범위 | 기준 |
