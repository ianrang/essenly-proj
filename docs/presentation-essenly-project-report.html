1. Executive Summary

1.1 프로젝트 정의

에센리(Essenly)는 한국을 방문하는 외국인 여성 여행객(20~40대)이 K-뷰티 관련 의사결정 — 제품 구매, 피부 시술, 살롱, 맛집, 문화 체험 — 을 AI 대화형 웹 앱을 통해 쉽고 정확하게 내릴 수 있도록 지원하는 서비스입니다.

1.2 핵심 문제

  • 언어 장벽 — 제품/시술 정보가 주로 한국어. 영문 리뷰·가이드 부족
  • 선택 과부하 — 수천 개 제품, 수백 개 클리닉에서 본인에게 맞는 것을 판별하기 어려움
  • 맥락 부재 — 피부 타입, 여행 일정, 예산에 맞는 통합 정보 없음
  • 신뢰 문제 — 광고성 추천과 진정한 추천의 구분 어려움

2. 요구사항 분석 & 벤치마킹

2.1 요구사항 도출 과정

고객사로부터 수령한 사업 계획서와 UX 프로토타입(HTML)을 바탕으로 다음 순서로 요구사항을 체계화했습니다.

1

소스 문서 분석

사업 계획서(PDF)에서 사업 정체성, 타겟 사용자, 수익 모델을 추출. UX 프로토타입(HTML)에서 화면 흐름, 카드 타입, 인터랙션 패턴을 분석

2

PRD 작성 (v1.0 → v2.2)

제품 요구사항(WHAT)을 6개 섹션으로 구조화. v2.0에서 이슈 20건 해소, 기술 혼입 19개소 제거. v2.2에서 중복 제거 및 최종 확정. 사용자 결정 12건 전체 해소

3

정합성 검증

PRD ↔ DB 스키마 간 불일치 6건(X-1~X-6) + PRD 이관 8건 해소. 데이터 모델이 모든 요구사항을 지원하는지 교차 검증

4

성공 지표 정의

6개 KPI 이벤트 + 5개 사용자 행동 이벤트 정의. 측정 방식 결정: 자체 로그(behavior_logs) + Vercel Analytics 보조

3. 서비스 설계 — 기획 & UX

3.1 사용자 흐름 (Chat-First MVP)

사용자가 서비스에 진입하면 대화를 통해 자연스럽게 추천을 받는 흐름을 따릅니다. 별도 폼이나 온보딩 절차 없이 바로 대화를 시작할 수 있습니다.

flowchart LR A["랜딩 페이지"] --> B["'Start chatting'\n버튼 클릭"] B --> C["데이터 보관\n동의 확인"] C --> D["AI 인사 +\n제안 질문 3개"] D --> E["대화 진행\n(정보 자연 수집)"] E --> F["추천 카드\n표시"] F --> G["구매/예약\n외부 링크"] E --> H["키트 추천\n(에센리 제품)"]
설계 원칙 — 사용자가 아무 정보를 입력하지 않아도 추천이 가능합니다 (VP-3). "지성 피부에 좋은 세럼 추천해줘"처럼 대화하는 것만으로 AI가 피부 타입과 고민을 파악하여 프로필에 자동 저장합니다. 대화가 이어질수록 추천 정확도가 올라갑니다.

5.1.3 데이터 가공 파이프라인

수집된 원본 데이터는 4단계를 거쳐 서비스에 사용 가능한 형태로 가공됩니다.

graph LR S1["1단계: 수집

1단계: 수집 — 카카오 API, 식약처 API, 브랜드 사이트, CSV 파일 등 다양한 출처에서 원본 데이터를 확보합니다.

2단계: AI 보강 — 원본 데이터를 AI가 처리합니다.

  • 번역: 한국어 제품명/설명을 6개 언어(영/일/중/스페인/프랑스)로 번역
  • 분류: 적합 피부타입(건성/지성/복합 등), 관련 고민(모공/주름/건조 등)을 자동 분류. AI의 확신도(confidence score)를 함께 산출하여 검수 우선순위 결정
  • 설명 생성: 제품 특징, 시술 과정, 주의사항 등을 한국어/영어로 자동 작성

3단계: 전수 검수 — AI 분류 결과를 사람이 전부 확인합니다. 확신도 50% 미만 항목을 우선 검토하며, 자동 승인 없이 검수자가 최종 승인해야 다음 단계로 진행됩니다.

4단계: DB 적재 + 벡터 생성 — 검수 완료된 데이터를 데이터베이스에 저장합니다. 각 제품/시술 정보를 숫자 벡터로 변환하여, 사용자 질문과 의미적으로 유사한 항목을 빠르게 찾을 수 있게 합니다.

AI 보강 비용: 200개 제품 기준 약 $5.50 (1회성 비용, 월간 비용 아님). 데이터 변경 시에만 추가 비용 발생.

5.2 AI 대화 서비스 — 사용자 질문에 어떻게 답하는가

5.2.1 사용자 관점의 대화 흐름

사용자 질문 예시: "지성 피부인데 모공 줄여주는 제품 있어?"
1

AI가 질문을 이해합니다

"이 사용자는 지성(oily) 피부이고, 모공(pores) 고민이 있다. 제품 검색이 필요하다." — AI가 질문의 의도를 파악하고, 어떤 기능을 사용할지 스스로 결정합니다.

2

데이터베이스에서 맞는 제품을 검색합니다

부적합 제품 자동 제외(조건 필터) → 남은 제품에 점수 부여(개인화 순위) → 높은 점수순으로 3~5개 선정. 상세 원리는 5.2.3 참조.

3

자연어 + 카드로 응답합니다

"지성 피부에 모공 관리용으로 3가지를 추천해드릴게요!" — 텍스트 응답이 한 글자씩 실시간 표시(스트리밍)되며, 추천 제품은 이미지·가격·추천 이유·구매 매장이 담긴 카드로 자동 렌더링됩니다.

4

뒷작업이 자동으로 실행됩니다 (사용자에게 보이지 않음)

대화 내용을 저장하고(다음 방문 시 이어서 대화 가능), "지성 피부" 정보를 프로필에 자동 저장합니다(다음 추천부터 반영).

5.2.2 AI 도구 — AI가 스스로 판단하여 사용하는 4가지 기능

AI는 사용자 질문에 따라 어떤 도구를 쓸지 자동으로 판단합니다. 한 번의 대화에서 필요한 만큼 여러 도구를 연속 사용할 수 있습니다.

도구언제 사용하나무엇을 하나
제품/시술 검색 추천을 요청받았을 때 DB에서 조건에 맞는 제품/시술을 찾고, 개인화 점수를 매겨서 카드로 반환
성분/시술 지식 조회 전문 정보가 필요할 때 성분 효능, 시술 과정 등 K-뷰티 지식 베이스에서 정보 참조
사용자 정보 추출 대화 중 개인 정보를 언급했을 때 피부타입, 예산, 체류기간 등을 자동 감지하여 프로필에 저장
외부 링크 생성 구매/예약 안내가 필요할 때 제품 구매처, 클리닉 예약 페이지 링크를 제공

5.2.3 개인화 추천의 작동 원리

추천은 "부적합 제품 제거 → 남은 제품 순위 매기기"의 2단계로 동작합니다.

1단계: 부적합 제품 자동 제거 (조건 필터)

사용자 프로필과 맞지 않는 제품을 결과에서 제외합니다. 이 단계에서 걸러진 제품은 사용자에게 보이지 않습니다.

조건동작예시
피부타입 불일치결과에서 제외지성 피부 사용자에게 "건성 피부 전용 크림"은 표시 안 함
고민과 무관결과에서 제외"모공" 고민인데 "주름 전용" 제품은 표시 안 함
예산 초과결과에서 제외예산 5만원인데 8만원 제품은 표시 안 함
회복기간 > 체류기간
(시술만 해당)		결과에서 제외3일 남았는데 회복 5일 필요한 시술은 표시 안 함

2단계: 남은 제품에 점수 매기기 (개인화 순위)

1단계를 통과한 모든 제품에 점수를 부여합니다. 어떤 제품도 추가로 제외하지 않으며, 점수에 따라 표시 순서만 결정됩니다.

기준점수 영향예시
사용자가 선호하는 성분 포함점수 상승 (+0.1/성분)"달팽이 점액 좋아요"라고 했으면 스네일 에센스가 높은 순위
사용자가 기피하는 성분 포함점수 하락 (-0.15/성분)"향 싫어요"라고 했으면 향료 포함 제품 순위 하락
질문과 의미적 유사도점수 상승"모공 줄이는 클렌저"와 의미적으로 가까운 제품이 상위
회복기간 경고
(시술만 해당)		점수 소폭 하락체류 5일 중 회복 3일이면 경고와 함께 순위 약간 하락
이 2단계 구조의 장점 — 1단계에서 확실히 안 맞는 제품을 제거하므로 사용자가 불필요한 결과를 볼 일이 없고, 2단계에서 미세한 선호도를 반영하므로 대화를 많이 할수록 결과가 정확해집니다.
광고 중립 원칙 (VP-1) — 에센리 제휴 제품/클리닉은 "하이라이트 배지"가 표시되지만, 추천 점수에는 영향을 주지 않습니다. 순위는 오직 사용자 프로필과 질문 내용에 의해서만 결정됩니다.

프로필이 없어도 작동합니다 (점진적 개인화)

  • 처음 방문: 모든 제품이 동일 점수에서 시작 → 질문과의 의미 유사도만으로 추천
  • 대화를 하면서: AI가 자연스럽게 피부타입, 고민, 예산 등을 파악하여 프로필에 저장
  • 다음 질문부터: 파악된 정보가 조건 필터와 점수 계산에 반영되어 더 정확한 추천

5.2.4 검색 방식 — 조건 검색과 의미 검색의 결합

제품 검색은 두 가지 방식을 결합하여 정확도를 높입니다.

정확 조건 검색의미 기반 검색
작동 방식피부타입, 예산, 카테고리 등 정확한 조건으로 필터링사용자 질문의 의미를 분석하여 가장 유사한 제품 찾기
장점100% 정확한 조건 매칭"모공 줄이는" 같은 자연어 표현 이해 가능
단독 한계"피곤한 피부에 활력"처럼 키워드와 다른 표현 이해 불가조건 필터 없이는 부적합 제품도 포함될 수 있음
결합 효과조건 필터로 부적합 제품을 먼저 제거한 후, 남은 제품 중에서 의미적으로 가장 유사한 순서로 정렬

의미 검색 서비스에 일시적 장애가 발생하면, 조건 검색만으로 자동 전환됩니다. 추천 품질은 다소 낮아질 수 있지만 서비스는 중단 없이 계속됩니다.

5.2.5 안정성 & 확장성 설계

설계내용
AI 모델 교체 가능환경변수 하나로 Claude ↔ Gemini 전환. 특정 AI 회사에 종속되지 않음
자동 폴백주 AI(Claude) 장애 시 자동으로 보조 AI(Gemini)로 전환. 사용자는 차이를 느끼지 못함
실시간 스트리밍응답을 한 글자씩 실시간 표시. 전체 응답을 기다리지 않아 체감 속도 향상
도메인 확장현재 쇼핑+시술 → 향후 살롱, 맛집, 문화체험 추가 시 동일한 구조에 플러그인
비용 통제대화당 토큰 사용량 기록. 일일/분당 요청 제한(Rate Limit)으로 비용 급증 방지
